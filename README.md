## Описание репозитория

Проект реализует полный минимальный пайплайн:

1. подготовка обучающего поднабора из датасета задания 3;
2. локальное дообучение LoRA-адаптера на базовой instruction-tuned модели;
3. генерация adversarial-датасета после обучения;
4. сохранение результатов в табличном формате для анализа и включения в отчет.

Целевой сценарий: генерация пользовательских запросов с шумом, опечатками, разрывами слов, неполными данными и конфликтными ограничениями, способных использоваться для тестирования устойчивости продуктового чат-бота.


## Использованная базовая модель

- `Qwen/Qwen2.5-1.5B-Instruct`

Модель выбрана как компромисс между качеством генерации и возможностью локального запуска в ограниченной вычислительной среде.


## Структура файлов

### `prepare_task4_dataset.py`
Подготавливает обучающий поднабор из Excel-датасета задания 3.

Что делает:
- читает исходный Excel;
- отбирает примеры, релевантные сценарию noisy input и смежным кейсам;
- преобразует данные в instruction-format;
- сохраняет train / validation / test в JSONL.


### `train_lora_task4.py`
Запускает LoRA fine-tuning базовой модели.

Что делает:
- загружает базовую модель;
- подключает LoRA-конфигурацию;
- читает подготовленный instruction-датасет;
- запускает supervised fine-tuning;
- сохраняет обученный адаптер.


### `generate_adversarial_dataset.py`
Использует базовую модель + обученный LoRA-адаптер для генерации adversarial-промптов.

Что делает:
- загружает обученный адаптер;
- формирует generation prompts;
- генерирует новые adversarial-запросы;
- сохраняет результат в Excel.


## Папки

### `data/`
Содержит исходные данные, включая Excel-датасет из задания 3

### `outputs/`
Содержит все артефакты пайплайна:
- JSONL-файлы train / validation / test;
- обученный LoRA-адаптер;
- сгенерированный adversarial-датасет;
- дополнительные промежуточные файлы.
